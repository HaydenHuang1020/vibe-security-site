25 项上线检查，覆盖真正会出事的地方

自动扫描负责发现线索，人工检查负责理解代码上下文、验证影响并排除误报。

01

支付安全

  • Stripe 密钥暴露

    识别前端代码与公开配置中的 secret key。

  • Stripe Webhook 签名

    确认事件使用原始请求体和 webhook secret 验签。

02

数据库安全

  • Supabase / Firebase 权限

    检查 RLS、开放策略和客户端敏感表访问。

  • SQL 注入

    寻找拼接查询与未参数化输入。

  • IDOR 越权访问

    验证资源 ID 查询是否同时校验所有权。

03

认证安全

  • JWT 算法与密钥

    检查 none 算法、弱密钥及不安全验证。

  • 密码存储

    识别明文比较和不安全存储。

  • 认证端点速率限制

    检查登录、注册和重置密码端点。

04

权限控制

  • 管理员 API

    确认后端校验身份与角色，而非只隐藏按钮。

  • CSRF 保护

    检查 Cookie、Origin 与 CSRF token 策略。

05

代码安全

  • XSS

    识别危险 HTML 注入和未经净化的用户内容。

  • 开放重定向

    检查来自用户输入的跳转目标。

  • 输入验证

    确认 API 对关键字段进行服务端验证。

  • 文件上传

    检查 MIME、大小、文件名和存储位置。

  • 硬编码密钥

    查找常见云服务、支付和仓库凭证格式。

06

信息泄露

  • .env 暴露

    识别包含真实凭证的环境文件。

  • 调试端点

    寻找生产环境遗留的测试和调试路由。

  • 日志敏感信息

    检查 token、密码和用户隐私数据输出。

  • 错误信息

    识别堆栈、查询和内部路径泄露。

  • Source Map

    检查生产构建是否暴露源代码映射。

07

基础设施

  • 安全响应头

    检查 CSP、HSTS、X-Frame-Options 等。

  • CORS

    识别过宽来源策略和凭证组合风险。

  • Cookie 安全

    检查 Secure、HttpOnly 和 SameSite。

  • 凭证文件

    识别私钥、服务账号和云凭证文件。

08

依赖安全

  • npm audit

    检查已知严重和高危依赖漏洞。

自动化检查不是安全结论

扫描结果会经过人工复核。我们会查看调用链、认证方式、部署配置和数据边界，确认问题是否真实可利用。

最终报告明确区分：已验证风险、需要配置确认的风险、误报，以及自动检查无法覆盖的人工检查项。
